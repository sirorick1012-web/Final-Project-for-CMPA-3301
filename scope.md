## 1. Project Purpose & Objectives

The purpose of this project is to design and configure a secure centralized Active Directory Domain Services (AD DS) infrastructure to streamline user authentication, resource management, and security policy enforcement across the organization.
Key objectives include:

    A. Creating user and group management.
	B. Group Policy Objects (GPOs) for security.
	C. Creation of Objects for devices (computers, laptops, ect...).
	  
## 2. Scope Description
	  
The project will include:

   A. Assessment of current IT infrastructure and requirements.
   B. Design of the AD DS architecture, including domain structure, OU hierarchy, and naming conventions.
   C. Creation of user, group, and service accounts.
   D. Implementation of Group Policies for security and compliance.
   E. Documenting 
   
## 3. Deliverables
   
      A. Fully functional AD DS environment.
	  B. Organizational Unit (OU) structure and Group Policies in place.
	  C. User and group accounts migrated or created.
	  
## 4. Exclusions

     A. Migration of applications and devices not compatible with AD.
	 B. End-user device upgrades outside of AD integration requirements.
	 
## 5. Constraints

    A. Implementation must be completed within the agreed project timeline and budget.
	B. All configurations must comply with organizational security policies and prcedures.
	
## 6. Assumptions

     A. IT staff will be available for training and testing phases.
